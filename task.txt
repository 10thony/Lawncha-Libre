DESIGN PRINCIPLES FOR TRANSPOSITION
------------------------------------
1. LOYAL TO THE POC: Do not invent new aesthetics. Copy the palette, typography,
   spacing, and border treatment from the proof-of-concept files exactly.

2. FUNCTIONAL PARITY: Every interactive element (buttons, selects, inputs, modals)
   must work identically across styles. Only the visual treatment changes.

3. FONT LOADING: Each style uses specific Google Fonts. Ensure the correct
   @import or <link> tags are present. Do not substitute fonts.
     - warroom:   JetBrains Mono
     - dossier:   Courier Prime, Special Elite, Libre Baskerville
     - brutalist:  Archivo Black, IBM Plex Sans
     - transit:    Chivo, Chivo Mono

4. NO GENERIC AI AESTHETICS: Avoid Inter, Roboto, purple gradients, rounded
   everything. Each style has a SPECIFIC personality. Respect it.

5. VERIFY WITH BROWSER: After every component transposition, open the browser
   tab and visually confirm the style matches the POC concept. Screenshot if needed.

6. EMBEDDED MODE: When a styled component renders inside the Dashboard content
   area, suppress any standalone navigation (top bars with route links).
   Use the `embedded` prop pattern established in the Projects POCs.
